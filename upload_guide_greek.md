# Οδηγός Ανεβάσματος Εκπαιδευτικού Υλικού

Αυτός ο οδηγός εξηγεί τη διαδικασία εισαγωγής νέων ερωτήσεων στη βάση δεδομένων του Chatbot. Η διαδικασία χωρίζεται σε δύο ρόλους: τον **Διαχειριστή Συστήματος (Admin)** και τον **Εκπαιδευτικό**.

---

## Μέρος Α: Προετοιμασία Αρχείου (Εκπαιδευτικός)

Πριν ανεβάσετε οτιδήποτε, το αρχείο (.docx ή .pdf) πρέπει να έχει τη σωστή μορφή για να το διαβάσει το ρομπότ.

**Κανόνες Μορφοποίησης:**
1.  **Ερωτήσεις:** Πρέπει να ξεκινούν με τη λέξη `Ερώτηση:` ή `Q:` ή `Θέμα:`
2.  **Απαντήσεις:** Πρέπει να ξεκινούν με τη λέξη `Απάντηση:` ή `A:` ή `Λύση:`

**Παράδειγμα:**
> **Ερώτηση 1:** Ποιος είναι ο χημικός τύπος του νερού;
> **Απάντηση:** Ο τύπος είναι H2O.
>
> **Θέμα 2:** Να περιγράψετε τον κύκλο του νερού.
> **Λύση:** Ο κύκλος του νερού αποτελείται από...

---

## Μέρος Β: Μεταφορά στον Server (Διαχειριστής)

Η μεταφορά του αρχείου στον Server γίνεται μέσω του **GitHub Desktop**, ώστε να υπάρχει ιστορικό αλλαγών και ασφάλεια.

**Βήματα (Στον Υπολογιστή του Διαχειριστή):**
1.  Αποθηκεύστε το αρχείο του εκπαιδευτικού (π.χ. `fysiki.docx`) μέσα στον φάκελο `files` του project στον υπολογιστή σας (`C:\RasaFinalProject\files`).
2.  Ανοίξτε το **GitHub Desktop**.
3.  Θα δείτε το νέο αρχείο στην λίστα αριστερά.
4.  Γράψτε ένα μήνυμα (π.χ. "Προσθήκη Φυσικής") και πατήστε **Commit to main**.
5.  Πατήστε το κουμπί **Push origin** (πάνω δεξιά) για να ανέβει στο Cloud.

**Βήματα (Στον Server):**
1.  Συνδεθείτε στον Server (SSH).
2.  Πηγαίνετε στον φάκελο του project:
    `cd ~/rasa-school-chatbot-v2`
3.  Κατεβάστε τις αλλαγές:
    `git pull`

*Τώρα το αρχείο βρίσκεται στον Server και είναι έτοιμο!*

---

## Μέρος Γ: Ενεργοποίηση από το Chatbot (Εκπαιδευτικός)

Τώρα ο εκπαιδευτικός μπαίνει στο Chatbot (μέσω της ιστοσελίδας) για να "πυροδοτήσει" την ανάγνωση.

**Προϋπόθεση:** Πρέπει να είστε **συνδεδεμένος** χρήστης (Member/Admin) στην ιστοσελίδα.

**Βήματα:**
1.  Ανοίξτε το Chatbot.
2.  Γράψτε την εντολή: **"Ανέβασμα υλικού"** ή **"Θέλω να ανεβάσω ασκήσεις"**.
3.  Το Bot θα ελέγξει αν είστε εγγεγραμμένος.
    *   *Αν όχι:* Θα σας ζητήσει να συνδεθείτε.
    *   *Αν ναι:* Θα ρωτήσει: *"Παρακαλώ δώστε μου το μονοπάτι του αρχείου"*
4.  Γράψτε το όνομα του αρχείου που ανέβασε ο διαχειριστής:
    **`fysiki.docx`**
5.  Το Bot θα διαβάσει το αρχείο, θα βρει τις ερωτήσεις/απαντήσεις και θα τις αποθηκεύσει στη βάση δεδομένων.
6.  Θα επιβεβαιώσει: *"Επιτυχία! Αποθηκεύτηκαν 15 ερωτήσεις..."*

## Συνοπτικό Διάγραμμα

1.  📄 **Εκπαιδευτικός:** Γράφει το Word/PDF.
2.  💻 **Διαχειριστής (PC):** Βάζει το αρχείο στο GitHub Desktop -> Push.
3.  ☁️ **Διαχειριστής (Server):** Κάνει `git pull`.
4.  🤖 **Εκπαιδευτικός (Chat):** Λέει "Ανέβασμα" -> "filename.docx".
5.  ✅ **Chatbot:** Αποθηκεύει τις ερωτήσεις.
