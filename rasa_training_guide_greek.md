# Οδηγός Εκπαίδευσης Rasa Chatbot

Αυτός ο οδηγός εξηγεί πώς λειτουργεί η εκπαίδευση του Rasa chatbot και πώς μπορείτε να προσθέσετε νέα γνώση βήμα προς βήμα.

## Βασικές Έννοιες

Το Rasa χρησιμοποιεί 3 κύρια αρχεία για να μάθει τι να κάνει:

1.  **`data/nlu.yml` (Natural Language Understanding)**
    *   Εδώ μαθαίνουμε στο bot να **καταλαβαίνει** τι λέει ο χρήστης.
    *   Ορίζουμε **Intents** (προθέσεις) και δίνουμε παραδείγματα.
2.  **`domain.yml`**
    *   Εδώ είναι το "σύμπαν" του bot.
    *   Ορίζουμε όλες τις πιθανές **απαντήσεις (Responses)** και καταγράφουμε τα Intents που φτιάξαμε.
3.  **`data/stories.yml`** ή **`data/rules.yml`**
    *   Εδώ μαθαίνουμε στο bot τη **ροή** της συζήτησης.
    *   Οι **Rules** είναι για απλούς κανόνες (Ερώτηση A -> Πάντα Απάντηση B).
    *   Τα **Stories** είναι για πιο περίπλοκους διαλόγους με ιστορικό.

---

## Βήμα 1: Προσθήκη NLU (`data/nlu.yml`)

Ανοίξτε το αρχείο `data/nlu.yml`. Εδώ θα προσθέσετε παραδείγματα για το τι μπορεί να ρωτήσει ο χρήστης.

**Μορφή:**
```yaml
- intent: onoma_neou_intent
  examples: |
    - ερώτηση χρήστη 1
    - ερώτηση χρήστη 2
    - άλλη διατύπωση της ερώτησης
```

**Παράδειγμα:**
Ας πούμε ότι θέλουμε το bot να απαντάει για το **ωράριο**.
```yaml
- intent: ask_opening_hours
  examples: |
    - τι ώρες είστε ανοιχτά;
    - πότε μπορώ να έρθω;
    - ποιο είναι το ωράριο λειτουργίας;
    - είστε ανοιχτά το απόγευμα;
```

---

## Βήμα 2: Προσθήκη Απάντησης (`domain.yml`)

Ανοίξτε το αρχείο `domain.yml`. Πρέπει να κάνετε δύο πράγματα εδώ:

1.  **Καταχώρηση του Intent**: Βρείτε τη λίστα `intents:` και προσθέστε το όνομα του νέου intent.
    ```yaml
    intents:
      - greet
      - goodbye
      - ask_opening_hours  # <-- ΝΕΟ
    ```

2.  **Δημιουργία Απάντησης (Response)**: Βρείτε την ενότητα `responses:` και προσθέστε την απάντηση. Οι απαντήσεις συνήθως ξεκινούν με `utter_`.
    ```yaml
    responses:
      utter_opening_hours:
        - text: "Είμαστε ανοιχτά καθημερινά 09:00 - 21:00 και το Σάββατο 09:00 - 14:00."
    ```

---

## Βήμα 3: Σύνδεση Ερώτησης με Απάντηση (`data/rules.yml`)

Για απλές ερωτήσεις-απαντήσεις (Q&A), το `data/rules.yml` είναι το καλύτερο μέρος.

Ανοίξτε το `data/rules.yml` και προσθέστε έναν κανόνα:

```yaml
- rule: Απάντηση για ωράριο
  steps:
  - intent: ask_opening_hours
  - action: utter_opening_hours
```

*Αυτό λέει στο bot: "Κάθε φορά (Always) που αναγνωρίζεις το intent `ask_opening_hours`, κάνε το action `utter_opening_hours`."*

---

## Βήμα 4: Εκπαίδευση (Training)

Αφού αποθηκεύσετε όλα τα αρχεία (`nlu.yml`, `domain.yml`, `rules.yml`), πρέπει να εκπαιδεύσετε το μοντέλο ξανά για να μάθει τις αλλαγές.

Ανοίξτε το τερματικό (ή την γραμμή εντολών) μέσα στον φάκελο του project και τρέξτε:

```bash
rasa train
```

Η διαδικασία θα πάρει λίγο χρόνο (ανάλογα με το μέγεθος των δεδομένων). Όταν τελειώσει, θα δείτε ένα μήνυμα ότι το νέο μοντέλο αποθηκεύτηκε στο φάκελο `models/`.

---

## Βήμα 5: Δοκιμή

Για να δοκιμάσετε το νέο σας bot, τρέξτε:

```bash
rasa shell
```

Αυτό θα ανοίξει μια συνομιλία στο τερματικό. Γράψτε μία από τις ερωτήσεις που βάλατε (π.χ., "τι ώρες είστε ανοιχτά;") και δείτε αν απαντάει σωστά.

## Συνοπτικό Checklist

1.  [ ] **NLU**: Προσθήκη παραδειγμάτων ερώτησης στο `data/nlu.yml`.
2.  [ ] **Domain**: Προσθήκη του intent και της απάντησης (`utter_...`) στο `domain.yml`.
3.  [ ] **Rules**: Σύνδεση intent με απάντηση στο `data/rules.yml`.
4.  [ ] **Train**: Εκτέλεση `rasa train`.
5.  [ ] **Test**: Δοκιμή με `rasa shell`.
