# Οδηγός Google Cloud Platform (GCP)

Αυτό το έγγραφο εξηγεί τα εργαλεία του Google Cloud που χρησιμοποιήσαμε για να βγάλουμε το Chatbot "στον αέρα" (Online Deployment).

---

## 1. Τι είναι το Google Cloud;
Το Google Cloud είναι μια πλατφόρμα που μας νοικιάζει υπολογιστική ισχύ. Αντί να τρέχει το Chatbot στον δικό σας φορητό υπολογιστή (που κλείνει το βράδυ), τρέχει σε έναν υπολογιστή της Google που είναι ανοιχτός 24/7.

---

## 2. Ποιες Υπηρεσίες Χρησιμοποιήσαμε;

### Α. Compute Engine (VM Instances)
*   **Τι είναι:** Είναι η "καρδιά" του συστήματος. Πρόκειται για μια Εικονική Μηχανή (Virtual Machine - VM), δηλαδή έναν υπολογιστή που τρέχει μέσα στους servers της Google.
*   **Γιατί το χρησιμοποιήσαμε:** Χρειαζόμασταν έναν υπολογιστή με λειτουργικό σύστημα **Linux (Ubuntu/Debian)** για να εγκαταστήσουμε το Docker και να τρέχουμε το Chatbot.
*   **Τεχνικά Χαρακτηριστικά:**
    *   *Machine Type:* Επιλέξαμε τύπο **e2-medium** (2 vCPUs, 4GB RAM).
    *   *Λόγος:* Το Rasa είναι "βαρύ" λογισμικό (Machine Learning) και χρειάζεται αρκετή μνήμη RAM (τουλάχιστον 4GB) για να εκπαιδεύεται και να απαντά γρήγορα. Τα μικρότερα μηχανήματα (micro) κολλούσαν.

### Β. VPC Network (External IP Addresses)
*   **Τι είναι:** Η διεύθυνση IP (π.χ. `34.123.45.67`) είναι η "ταυτότητα" του server στο ίντερνετ.
*   **Γιατί το χρησιμοποιήσαμε:** Αγοράσαμε (δεσμεύσαμε) μια **Static External IP**.
*   *Λόγος:* Αν δεν είχαμε στατική IP, κάθε φορά που γινόταν επανεκκίνηση του server, η διεύθυνση θα άλλαζε και το Chatbot θα "εξαφανιζόταν" από το site του σχολείου. Με τη στατική IP, το site ξέρει πάντα πού να το βρει.

### Γ. Firewall Rules (Τείχος Προστασίας)
*   **Τι είναι:** Κανόνες ασφαλείας που λένε ποιος επιτρέπεται να μιλήσει με τον server.
*   **Γιατί το χρησιμοποιήσαμε:** Από προεπιλογή, η Google μπλοκάρει τα πάντα για ασφάλεια. Εμείς δημιουργήσαμε κανόνες για να επιτρέψουμε:
    *   **Port 80 (HTTP) & 443 (HTTPS):** Για να μπορεί ο κόσμος (και το site του σχολείου) να βλέπει το Chatbot μέσω Web.
    *   **Port 22 (SSH):** Για να μπορούμε εμείς (οι διαχειριστές) να συνδεθούμε για ρυθμίσεις.

### Δ. SSH (Secure Shell)
*   **Τι είναι:** Το "μαύρο παράθυρο" (τερματικό) που χρησιμοποιούμε για να δίνουμε εντολές στον server.
*   **Πώς το χρησιμοποιούμε:** Μέσω του browser (κουμπί "SSH" στο Google Cloud Console) ή από το τερματικό του υπολογιστή μας. Είναι κρυπτογραφημένη σύνδεση για μέγιστη ασφάλεια.

---

## 3. Συνοψίζοντας τη Διαδικασία

1.  Δημιουργήσαμε το **VM Instance** (τον υπολογιστή).
2.  Ρυθμίσαμε το **Static IP** και ανοίξαμε τις πόρτες στο **Firewall**.
3.  Συνδεθήκαμε με **SSH**.
4.  Εγκαταστήσαμε το **Docker** (το εργαλείο που τρέχει το Chatbot).
5.  Κατεβάσαμε τον κώδικα από το **GitHub** (`git clone`).
6.  Ξεκινήσαμε την εφαρμογή (`docker-compose up`).

Πλέον, το Google Cloud φροντίζει ώστε αυτός ο υπολογιστής να έχει πάντα ρεύμα, ίντερνετ και ασφάλεια, εξυπηρετώντας τους μαθητές οποιαδήποτε ώρα και στιγμή!
